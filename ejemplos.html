4.1. Aspectos generales del software

4.1.1. Ejemplo 1

Pregunta: ¿Cómo define el IEEE el software y cuáles son los elementos que incluye esta definición en relación con las operaciones de un sistema de cómputo?

Respuesta: El IEEE define el software como "el conjunto de los programas de computación, procedimientos, reglas, documentación y datos asociados que forman parte de las operaciones de un sistema de cómputo". Esta definición abarca no solo los programas ejecutables, sino también los procedimientos, normas, documentación y datos necesarios para el funcionamiento del sistema.

4.1.2. Ejemplo 2

Pregunta: ¿Cuáles son los tres componentes principales del software y cómo contribuyen al funcionamiento de un sistema?

Respuesta:

  • Programas: Instrucciones ejecutables que proporcionan funcionalidad y rendimiento.
  • Estructuras de datos: Permiten la manipulación eficiente de la información.
  • Documentación: Información explicativa (guías, manuales) para el uso y mantenimiento.

Adicionalmente, el software incluye archivos de configuración, soporte técnico y actualizaciones.

4.1.3. Ejemplo 3

Pregunta: ¿Cómo clasifica Roger Pressman el software según su propósito?

Respuesta: Pressman identifica siete categorías:

  • Software de sistemas: Sistemas operativos, compiladores (interacción directa con hardware).
  • Software de aplicación: Soluciones específicas (ERP, gestión empresarial).
  • Científico/ingeniería: Cálculos complejos y simulaciones.
  • Empotrado: Controla dispositivos físicos (electrodomésticos, automóviles).
  • Línea de productos: Software comercial (hojas de cálculo, procesadores de texto).
  • Aplicaciones web: Servicios en línea (banca, comercio electrónico).
  • Inteligencia artificial: Sistemas expertos, redes neuronales.

4.2. Ingeniería de Software

4.2.1. Ejemplo 5

Pregunta: ¿Qué es la ingeniería de software y cuáles son sus capas esenciales?

Respuesta: Es una disciplina que aplica principios de ingeniería al desarrollo de software, asegurando calidad, eficiencia y mantenibilidad.

Capas:

  • Compromiso con la calidad: Filosofías como Six Sigma.
  • Procesos: Estructuras para gestión de proyectos (ej: Agile).
  • Métodos: Técnicas para análisis, diseño y pruebas.
  • Herramientas: Soporte automatizado (IDE, sistemas CASE).

Autores clave: Bauer (1972), Sommerville (2004), Pressman (2005).

4.2.6. Ejemplo 10

Pregunta: ¿Cómo difiere la ingeniería de software de otras disciplinas?

Respuesta:

  • Naturaleza intangible: No hay producto físico; el "desgaste" es por obsolescencia o cambios técnicos.
  • Mantenimiento: Corrección de errores y adaptación a nuevos entornos (80% del esfuerzo).
  • Metodologías: Requiere flexibilidad debido a la evolución constante (ej: Scrum vs. enfoques rígidos).
